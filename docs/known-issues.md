# 既知事項と次期対応候補

README の既知事項を、開発・運用判断向けに詳細化します。

## 1. CSS 内の `rgb()` が `rgb（...）` のように全角括弧化する問題

### 現象

HTML 内の CSS 値に含まれる `rgb()` が、表記正規化や周辺処理の影響で `rgb（...）` のように全角括弧へ変換される場合があります。

### 影響

CSS として無効な値になり、該当箇所の色指定などのスタイルが反映されない可能性があります。本文テキストや table 構造そのものを破壊する問題ではありませんが、見た目の再現性に影響する可能性があります。

### 現時点の扱い

v1.0 では既知事項として扱います。社内向け移行品質では、本文欠落防止と table / iframe のアクセシビリティ補正を優先します。

### 次期対応方針

- CSS / style 属性内の関数記法を表記正規化対象から除外する。
- `style` 属性、`style` 要素、CSS 関数値を保護する前処理を追加する。
- 回帰検証に `rgb(` が `rgb（` へ変換されていないことを確認する項目を追加する。

## 2. caption id の重複可能性

### 現象

table caption に `id` を付与する処理で、ページ内に複数 table が存在する場合に同一 `id` が生成される可能性があります。

### 影響

HTML の妥当性の観点で、同一ページ内の `id` 重複は望ましくありません。支援技術やアンカー参照、テストコードが `id` を前提にする場合、意図しない caption を参照する可能性があります。

### 現時点の扱い

v1.0 では既知事項として扱います。caption 自体の存在と table の見出し情報付与を優先し、`id` 一意性の完全保証は次期対応候補とします。

### 次期対応方針

- ページ内 table の DOM 順インデックスを使って caption `id` を一意化する。
- 既存 `id` がある場合は保持し、新規付与時のみ重複チェックを行う。
- caption `id` と table 側の関連属性を付与する場合は、同一 DOM 内で整合するように生成する。
- 回帰検証に `//*[@id]` の重複検出を追加する。

## 3. table header row / col / none 判定がヒューリスティックであること

### 現象

thead が無い table について、先頭行を列見出しにするか、先頭列を行見出しにするか、または安全側で変換しないかを、ルールベースのスコアリングで判定しています。

### 影響

表の構造や文言によっては、実際の意味とは異なる見出し方向を選ぶ可能性があります。一方で、誤変換を避けるために `none` を選択した場合、アクセシビリティ補正が十分に行われない可能性があります。

### 現時点の扱い

v1.0 ではヒューリスティック判定を採用します。明確に判断できる table は補正し、判断が難しい table は安全側で未変換にします。セル文言は変更しません。

### 次期対応方針

- 判定スコアと根拠をログに残し、誤判定例を収集する。
- 自治体ページの代表的な table パターンを回帰検証ケースに追加する。
- 人手確認が必要な table を `needs_review` 相当として扱う運用を検討する。
- 将来的には old / ai / gold の成果物管理と組み合わせ、AI 補正結果を人が確認できる流れに接続する。
